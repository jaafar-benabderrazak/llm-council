# üìÑ Results-Only Article Generator Guide

## Overview

Two tools to generate **results-focused articles** that exclude detailed round-by-round discussions:

1. **`extract_results.py`** - Extract results from existing Markdown articles (Recommended ‚≠ê)
2. **`generate_results_article.py`** - Generate results from debate JSON files

---

## Tool 1: Extract Results (Recommended ‚≠ê)

### Usage

```bash
# From existing article
python extract_results.py article_20251227_004758.md

# With custom output name
python extract_results.py article_20251227_004758.md --output my_results.md
```

### What It Does

- ‚úÖ Extracts only results, conclusions, and findings
- ‚úÖ Removes all round-by-round discussions
- ‚úÖ Keeps executive summary, analysis, references
- ‚úÖ Reduces file size by ~80-90%
- ‚úÖ Clean, professional output

### Example Output

```
Original article: 40,806 characters
Results article:   5,430 characters
Reduction:         87%
```

### Sections Extracted

‚úÖ **Included:**
- Executive Summary
- Introduction
- Key Findings & Analysis
- Source Validation
- Verified References
- Consensus & Disagreements
- Common Misconceptions
- Technical Details
- Best Practices
- Recommendations
- Limitations & Gaps
- Future Outlook
- Conclusion

‚ùå **Excluded:**
- Round 1, 2, 3, etc. discussions
- Individual agent responses
- Detailed debates
- Back-and-forth exchanges

## Tool 2: Generate from JSON

### Usage

```bash
# From debate JSON
python generate_results_article.py debate_20251227_010458.json

# With custom output
python generate_results_article.py debate_20251227_010458.json --output results.md
```

### What It Does

- Extracts synthesis from JSON
- Parses into structured sections
- Generates clean Markdown

**Note**: Works best when debate has comprehensive synthesis. If synthesis is short, use `extract_results.py` instead.

## Quick Start Examples

### Example 1: Extract from Latest Article

```bash
# Find latest article
$latest = Get-ChildItem -Filter "article_*.md" | Sort-Object LastWriteTime -Descending | Select-Object -First 1

# Extract results
python extract_results.py $latest.Name
```

### Example 2: Batch Process Multiple Articles

```bash
# PowerShell
Get-ChildItem -Filter "article_*.md" | ForEach-Object {
    Write-Host "Processing: $($_.Name)"
    python extract_results.py $_.Name
}
```

### Example 3: Python Batch

```python
from pathlib import Path
import subprocess

for article in Path('.').glob('article_*.md'):
    print(f"Processing: {article.name}")
    subprocess.run(['python', 'extract_results.py', str(article)])
```

## Troubleshooting

### Issue 1: No Sections Extracted

**Problem**: Results article is very short

**Solution**: 
- Article may not have structured sections
- Use full article instead
- Or manually review synthesis

### Issue 2: Encoding Errors (Windows)

**Problem**: `UnicodeEncodeError`

**Solution**:
```powershell
$env:PYTHONIOENCODING="utf-8"
python extract_results.py article.md
```

### Issue 3: Missing Sections

**Problem**: Expected sections not in results

**Solution**:
- Check original article has those sections
- Sections may use different naming
- Edit `priority_sections` in script
